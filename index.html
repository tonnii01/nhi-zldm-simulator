NHI-ZLDM Architecture Overview

The three layers of the Zero-Trust Lifecycle Defense Model

Layer 1

Proactive Hardening

Neutralizes risk at the source before credentials go live.

  • Secrets Vault Integration No plaintext secrets in settings or configs
  • Privilege Justification Enforce minimum permissions before activation
  • CI/CD Secrets Scan Pre-commit scanning blocks leaked strings
Layer 2

Active Governance

Manages active identities with absolute authorization.

  • Inventory Registry Unregistered NHI services blocked automatically
  • Just-In-Time (JIT) Short-lived tokens that auto-expire on task end
  • Auto-Rotation Engine Automated rotation eliminates long-lived keys
  • Decommission Trigger Project retirement automatically revokes keys
Layer 3

Threat Containment

Identifies and neutralizes active compromises.

  • Behavioral Baselines Learns regular access frequencies and IPs
  • Anomaly Scoring Calculates risk scores for sudden activity spikes
  • Containment Playbook Tiered reactions: Alert, Throttle, or Suspend
